## P2：发布者体验补强

目标：让发布者工作流更像产品，而不是一组脚本命令集合。

- [ ] 提供模板中心，覆盖 UI、交互、数据驱动、特效等常见起点。
- [ ] 提供“人话报错”，错误至少包含：问题、位置、修复建议。
- [ ] 提供最小可复制样例集合，优先服务新发布者上手。
- [ ] 把积木协议的“标准化问题类型”固定下来，统一收敛为空间适配、交互适配、状态适配、资源绑定、约束冲突五类问题，作为后续参数化和校验的基础语义。
- [ ] 把高频微调项从隐式代码修改收敛为显式参数入口，优先明确空间、交互、状态、资源、约束相关参数，不在第一阶段追求“所有内容都参数化”。
- [ ] 先定义积木协议的六层标准化结构：空间层、交互层、状态层、逻辑层、资源层、约束层，并在契约文档中固定每层必填声明。
